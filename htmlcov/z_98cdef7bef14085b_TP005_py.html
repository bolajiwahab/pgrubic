8class Varchar(linter.BaseChecker): 

9 """## **What it does** 

10 Checks for usage of varchar. 

11 

12 ## **Why not?** 

13 varchar(n) is a variable width text field that will throw an error if you try and 

14 insert a string longer than n characters (not bytes) into it. 

15 

16 varchar (without the (n)) or text are similar, but without the length limit. 

17 If you insert the same string into the three field types they will take up exactly 

18 the same amount of space, and you won't be able to measure any difference 

19 in performance. 

20 

21 If what you really need is a text field with an length limit then varchar(n) is 

22 great, but if you pick an arbitrary length and choose varchar(20) for a surname 

23 field you are risking production errors in the future when Hubert Blaine 

24 Wolfe­schlegel­stein­hausen­berger­dorff signs up for your service. 

25 

26 Some databases do not have a type that can hold arbitrary long text, or if they do 

27 it's not as convenient or efficient or well-supported as varchar(n). 

28 Users from those databases will often use something like varchar(255) when what 

29 they really want is text. 

30 

31 If you need to constrain the value in a field you probably need something more 

32 specific than a maximum length - maybe a minimum length too, or a limited set of 

33 characters - and a check constraint can do all of those things as well as a maximum 

34 string length. 

35 

36 ## **When should you?** 

37 When you want to, really. If what you want is a text field that will throw an error 

38 if you insert too long a string into it, and you don't want to use an explicit check 

39 constraint then varchar(n) is a perfectly good type. Just don't use it automatically 

40 without thinking about it. 

41 

42 Also, the varchar type is in the SQL standard, unlike the text type, so it might be 

43 the best choice for writing super-portable applications. 

44 

45 ## **Use instead:** 

46 1. text 

47 2. text with a constraint that enforces a maximum string length 

48 """
